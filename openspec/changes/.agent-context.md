### specs/<capability>/spec.md
```
## ADDED Requirements

### Requirement: <name>
<requirement text with SHALL/MUST>

#### Scenario: <name>
- **WHEN** <condition>
- **THEN** <outcome>
```

**CRITICAL RULES:**
- Scenarios MUST use exactly 4 hashtags (`####`). Using 3 hashtags or bullets will fail silently.
- Every requirement MUST have at least one scenario.
- Use SHALL/MUST for normative requirements (avoid should/may).
- One spec file per capability, path = `specs/<capability-name>/spec.md` (kebab-case).

### tasks.md
```
## 1. <Task Group>
- [ ] 1.1 <task>
- [ ] 1.2 <task>

## 2. <Task Group>
- [ ] 2.1 <task>
```
- Tasks must be checkbox format: `- [ ] N.M description`
- Order by dependency
- Each task small enough for one session
- Include verification/validation tasks at the end
