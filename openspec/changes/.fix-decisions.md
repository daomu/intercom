This file is the single source of truth for all audit fix decisions. Each subagent reads this file and applies only the decisions tagged for its change.

Format: `### D<n>: Title (affects: XX, YY)` → decision + what to change in each affected change.

## Cross-cutting API canonical decisions

### D1: Packet encode/parse function names (affects: 08, 09, 10, 11, 12)
Canonical names: `encode_voice`/`parse_voice`, `encode_heartbeat`/`parse_heartbeat`, `encode_talk_state`/`parse_talk_state`, `encode_ctrl`/`parse_ctrl`, `encode_pair_beacon_host`/`parse_pair_beacon_host`, `encode_pair_join_req`/`parse_pair_join_req`, `encode_pair_join_ack`/`parse_pair_join_ack`, `encode_directory_broadcast`/`parse_directory_broadcast`, `encode_channel_switch_ack`/`parse_channel_switch_ack`.
- Change 10: rename all `build_voice_pkt` → `encode_voice`, `build_talk_state_pkt` → `encode_talk_state`, etc.
- Change 09: ensure all encode/parse call sites use canonical names.
- Change 11, 12: ensure parse call sites use canonical names.

### D2: Network method names (affects: 09, 10, 12)
Canonical: `send_broadcast(&self, payload: &[u8]) -> Result<(), NetError>` and `send_unicast(&self, dst: &[u8; 6], payload: &[u8]) -> Result<(), NetError>`. NO generic `send()`.
- Change 09: fix `network.send(BROADCAST, payload)` → `send_broadcast(payload)`; `send(mac, payload)` → `send_unicast(dst, payload)` in all tasks/design/proposal.
- Change 10, 12: verify call sites use `send_unicast`.

### D3: State enum — use IntercomState from change 08 (affects: 09, 10, 12, 13)
Canonical: `IntercomState` (Idle / Hosting(HostPhase) / Joining(JoinPhase) / Grouped(VoiceState)). `HostPhase` variants: Discovering, CollectingPeers, Frozen, SwitchingChannel. Use `IntercomState::Idle` for ungrouped (NOT `Ungrouped`).
- Change 09: DELETE the separate `PairingState` enum. Use `IntercomState`/`HostPhase`/`JoinPhase` directly. If data-carrying variants needed (e.g., members list), extend `HostPhase` with fields: `CollectingPeers { members: Vec<PeerEntry> }`, `Frozen { members: Vec<PeerEntry>, target: u8, offset: u16 }`, `SwitchingChannel { target: u8, deadline: Instant }`.
- Change 10, 12, 13: verify they reference `IntercomState::Idle` (not `Ungrouped`).

### D4: Battery 4-bar mapping — single source in PowerService (affects: 02, 04, 15)
Single source of truth: `PowerService::battery_step()` in change 04. BSP (02) ONLY reads raw ADC value, does NOT map to bars or percentages.
Voltage-based mapping (with ±0.1V hysteresis):
- < 3.4V → 1 bar (critical)
- 3.4–3.6V → 2 bars (low)
- 3.6–3.9V → 3 bars (medium)
- > 3.9V → 4 bars (full)
- Change 02: REMOVE all battery step/percentage mapping from BSP spec/design/tasks. BSP provides `read_raw_adc() -> u16` only. Remove D6 battery mapping, remove spec "电池档位" requirement, remove tasks 7.4 voltage thresholds.
- Change 04: own full mapping. Add spec requirement: "battery_step() SHALL map raw ADC to voltage via BoardProfile::BAT_ADC_DIVIDER (×3), then to 4-bar via thresholds <3.4/3.6/3.9V with ±0.1V hysteresis."
- Change 15: do NOT create separate BatterySampler. Call `PowerService::battery_step()` directly.

### D5: Button event model — BSP raw, Service classifies (affects: 02, 04)
BSP (02) emits RAW GPIO edge events only (no short/long/debounce classification, no 4th task). InputService (04) owns ALL classification via timer-based debouncing on Task C.
- BOOT (GPIO9): BSP emits `BootGpioPress` / `BootGpioRelease` (raw edges). InputService: hold ≥50ms then release → `BootPress` (PTT); release <50ms → `BootShortTap` (wake).
- PLUS (GPIO18): BSP emits `PlusGpioPress` / `PlusGpioRelease`. InputService: <500ms → `PlusShortPress`; ≥500ms → `PlusLongPress`.
- PWR: BSP MUST add PWR GPIO driver (interrupt-based), emits `PwrGpioPress` / `PwrGpioRelease`. InputService: <2s → `PowerShortPress`. Long press handled by ESP-IDF bootloader (hardware shutdown).
- Change 02: remove all short/long/debounce task from BSP. Remove tasks 6.1 (debounce task), 6.2 (200ms/600ms thresholds). Emit raw `*GpioPress`/`*GpioRelease` events. ADD PWR GPIO driver requirement + tasks.
- Change 04: own all threshold classification. Update spec to show BSP raw events → InputService classification pipeline.

### D6: PA control ownership (affects: 02, 05, 15)
- BSP (02): `pa_enable(on: bool)` = pure GPIO toggle (set PA_CTRL pin high/low). NO timing, NO delay, NO soft-start logic.
- AudioService (05): `start_playback`/`stop_playback` own soft-start sequence: `pa_enable(true)` → wait 1-2 frames buffer → start I2S output; stop: output zero frames 1-2 frames → `pa_enable(false)` → stop I2S.
- Change 15: PaController calls `AudioService::start_playback()`/`stop_playback()`, does NOT call `pa_enable()` directly. Remove PaController's own 20ms/50ms timing.
- Change 02: update spec to say `pa_enable` is pure GPIO toggle.
- Change 05: update spec to own soft-start sequence.

### D7: Mixing responsibility — AudioService owns mixing (affects: 05, 11)
- Jitter (11): decodes per-sender PCM, calls `AudioService::submit_pcm(sender_id, &pcm)` PER SENDER. NO attenuation/mixing in jitter.
- AudioService (05): `submit_pcm` owns mixing internally: fixed attenuation 0.7 per route → sum → soft limiter. Cap 3 active routes.
- Change 11: REMOVE all mixing/attenuation/limiter logic from spec/design/tasks. Jitter only: receive → dedup → jitter buffer → decode (via AudioService) → submit_pcm per sender.
- Change 05: update `submit_pcm` spec to document internal mixing pipeline.

### D8: Opus decoder ownership — AudioService exposes decode (affects: 05, 11)
- AudioService (05): add trait method `fn opus_decode(&self, frame: Option<&[u8]>) -> Result<[i16; 320], AudioError>`. `None` = PLC (predict lost frame). `Some(data)` = decode normal frame.
- Jitter (11): calls `audio_service.opus_decode(...)` instead of owning its own decoder. Remove Opus decoder instance from jitter.
- Change 05: add `opus_decode` to trait + spec.
- Change 11: update all decode call sites to use `audio_service.opus_decode()`.

### D9: switch_offset replaces switch_ts (affects: 08, 09)
DIRECTORY_BROADCAST: replace `switch_ts: u32` (4 bytes, absolute mono ms) with `switch_offset: u16` (2 bytes, relative ms from packet receipt).
- Join schedules `now_join + switch_offset` upon receiving first DIRECTORY_BROADCAST. Subsequent redundant broadcasts ignored (or refresh timer if offset already elapsed).
- Saves 2 bytes per packet, eliminates clock domain mismatch.
- Change 08: update DIRECTORY_BROADCAST spec — offset 11-12 = switch_offset (u16 big-endian), total payload shrinks by 2 bytes (was 7+N×38, now 5+N×38; 4 nodes = 5+152=157 payload, 165 total).
- Change 09: update pairing logic to use relative offset.

### D10: GroupInfo + last_state (affects: 03)
Add `pub last_state: u8` to `GroupInfo` struct (stores last foreground App id for PRD §5.2 restore). Update spec "数据结构定义" requirement. Update NVS `group` namespace to include `last_state` key (already in tech design §7.3). Update `save_group`/`load_group` scenarios to include this field.

### D11: Host cancel — Frozen cannot cancel (affects: 09)
Fix spec to match tasks: cancel is allowed ONLY in `Discovering` / `CollectingPeers`. `Frozen` state CANNOT cancel (must complete phase 2/3 or timeout). Update spec scenario "用户主动取消" to remove Frozen.

### D12: Task model compliance (affects: 02, 12)
- Change 02: NO 4th FreeRTOS task. BSP uses GPIO interrupt handlers (ISRs) that push raw edge events to a queue. InputService (on Task C) consumes the queue and does debouncing. Remove tasks 6.1 (debounce task creation).
- Change 12: heartbeat uses a periodic timer callback integrated into Task B's NetworkService recv dispatch loop. NOT `std::thread::spawn`. Update design D3 and tasks 2.3.

### D13: evaluate_channels return type (affects: 06)
Change `fn evaluate_channels(&self, candidates: &[u8]) -> u8` → `fn evaluate_channels(&self, candidates: &[u8]) -> Result<u8, NetError>`. Returns `Err(NetError::Channel)` when radio priority == P0. Update spec "NetworkService trait 定义" and "无线资源优先级 guard" requirement.

### D14: AudioFrame size 160 (affects: 05)
Change `MAX_OPUS_FRAME_SIZE` from 128 to 160. Update spec "AudioFrame 结构" requirement. Provides safety headroom for Opus VBR peaks.

### D15: Packet length corrections (affects: 08)
- PAIR_BEACON_HOST payload = **42 bytes** (6+32+1+1+1+1), NOT 50. The "50" was confusing total packet (8+42=50) with payload. Fix spec.
- DIRECTORY_BROADCAST total = **165 bytes** for 4 nodes (8 header + 5 payload-header + 4×38 entries = 8+5+152=165), using D9's 2-byte offset. Fix design risk calculation.
- PAIR_JOIN_ACK payload = 46 bytes (correct, no change).
- PAIR_JOIN_REQ payload = 44 bytes (correct, no change).

### D16: schema_ver truncation — tech debt (affects: 16)
Acknowledge in design.md that header `ver` field is 1 byte but `schema_ver` is `u16`. Mitigation: if schema_ver ever exceeds 255, must add a separate `ver_negotiation` packet type. For now (ver=1), no issue. Add to "Open Questions" or "Tech Debt" section.

### D17: Safe boot → Launcher guard (affects: 07, 16)
- Change 07: ADD spec requirement "安全启动模式守卫": if `safe_boot_flag` is set in DiagInfo, Launcher SHALL only display Settings App, SHALL skip IntercomService/NetworkService initialization, SHALL show a "安全模式" indicator.
- Change 16: document that `safe_boot_flag` is read by Launcher (change 07). Add cross-reference.

### D18: RecvEvent.payload — no heap alloc (affects: 06)
Change `payload: Vec<u8>` → `payload: heapless::Vec<u8, 250>` (ESP-NOW max payload ~250 bytes). Update spec "RecvEvent 结构" requirement. Add `heapless` to dependencies in proposal Impact section.

### D19: PacketHeader parse — magic only, ver deferred (affects: 08, 09)
`PacketHeader::parse` checks `magic == 0xC6` only. Returns parsed header (including `ver`) for caller to check. Does NOT reject on ver mismatch.
- Pairing layer (09): caller checks `ver != SCHEMA_VER` for PAIR_JOIN_REQ → sends `PAIR_JOIN_ACK(accepted=0, reason=2=incompatible)`.
- Non-pairing layer: caller checks ver, silently drops mismatched packets.
- Change 08: update spec "通用包头布局" to say parse checks magic only. Remove "ver != SCHEMA_VER → reject" from parse.
- Change 09: add ver check logic in pairing layer before sending JOIN_ACK.

### D20: ptt_release order (affects: 10)
Fix to match tech design §19.4: `stop_capture()` → `pa_enable(false)` → `send TALK_STATE(action=0)` (clear mode only) → `state = Grouped(Idle)`. Update spec "PTT release 流程" requirement.

### D21: Arbitration — Condvar not sleep (affects: 10)
Replace `std::thread::sleep(50ms)` with `Condvar::wait_timeout(50ms)`. ESP-NOW receive callback (or Task A recv path) parses incoming TALK_STATE during the window and sets `conflict_flag` + `condvar.notify()`. If notified with conflict → ChannelBusy. If timeout → no conflict → enter Talking. Update design D2 and tasks 3.2.

### D22: DIRECTORY_BROADCAST non-blocking (affects: 09)
Replace `sleep(80ms) × 5` with timer-based scheduling: schedule 5 broadcasts at `now + 0, 80, 160, 240, 320 ms` via a timer queue. Task B remains free to process incoming ACKs/JOIN_REQs between broadcasts. Update tasks 4.4.

### D23: set_radio_priority — deliberate addition (affects: 06)
Update spec to NOT claim "11 methods match §3.3". Instead state: "NetworkService trait extends tech design §3.3's 10 methods with `set_radio_priority` (D-guard for PRD §19.2 radio resource management). This is a deliberate addition, not a deviation." Update design D-guard.

### D24: Heartbeat Task B timer (affects: 12)
Design D3: heartbeat uses `esp_idf_svc::timer::EspTaskTimer` or a periodic `Instant`-based check in Task B's main loop. NOT `std::thread::spawn`. Update design D3, tasks 2.3. The heartbeat tick runs as part of Task B's event loop, checking `now - last_heartbeat >= period` each iteration.

### D25: Battery sampling single source (affects: 04, 15)

- Change 15: REMOVE separate BatterySampler. ScreenPolicy/StandbyPolicy call `PowerService::battery_step()` / `battery_level()` directly. Update design D7, tasks group 3.

### D27: PAIR_JOIN_ACK reason codes (affects: 09, 16)
Define canonical reason codes:

- 1 = Full (满员)
- 2 = SchemaIncompatible (不兼容)
- 3 = StateChanged (状态变化)
- Change 09: update spec to reference these codes.
- Change 16: update spec "跨设备 schema_ver 校验" to use `reason=2`.

### D28: About page reset reason — previous boot (affects: 16)
About page shows `DiagInfo.last_reset_reason` (stored at boot start = previous boot's reason). NOT `PowerService::reset_reason()` (current boot). At boot start, store `PowerService::reset_reason()` → `DiagInfo.last_reset_reason` in NVS. Update design D8, tasks 5.2.

### D29: inc_abnormal_boot conditional (affects: 03, 16)
`inc_abnormal_boot()` only called if `PowerService::reset_reason() != PowerOn`. Normal power-on does NOT increment. Update change 03 spec (method still exists, caller decides) and change 16 design/tasks (conditional call).

### D30: VoiceEffect from change 08, not redefined (affects: 08, 14)
- Change 14: do NOT redefine `VoiceEffect` enum in `src/intercom/voice_effect.rs`. USE the enum from change 08's `src/intercom/state.rs`.
- Change 08: ADD `to_u8(&self) -> u8` and `from_u8(u8) -> Option<VoiceEffect>` methods to `VoiceEffect` (Normal=0, PitchUp=1, PitchDown=2). Update spec "VoiceEffect 枚举" requirement. Add tasks for these methods.

### D31: Preview streaming — chunk processing (affects: 14)
Replace 3s full-buffer preview with streaming: capture 20ms frame → pitch shift in place → submit to playback queue → repeat for 3s (150 frames). Peak memory = ~640B per frame + 1 playback buffer (~640B), NOT 96KB. Update design D3, tasks group 4.

### D32: 3/4-device tests deferred (affects: 13, 17)
- Change 13: mark 3-device and 4-device layout tests as "deferred to integration (change 17), best-effort with 2 devices + code review". Update tasks group 10.
- Change 17: mark 3-device mixing and 4-device stress as "best-effort: 2 devices + code analysis; full 4-device validation deferred to future hardware availability". Update tasks group 8.

### D33: static mut → heapless/static+Mutex (affects: 11)
Replace all `static mut` in tasks with either `heapless::spsc::Queue` wrapped in a `static` (non-mut) + `CriticalSection` (from `critical-section` crate), or `static` + `Mutex`. Update tasks 2.3 (JitterPool), 2.2 (ring buffer).

### D34: spec scenario coverage for tech design §21 (affects: 17)
Add scenarios for ALL 10 tech design §21 items in spec "技术设计 §21 待验证项实机验证" requirement. Currently only 4 scenarios — add: Opus CPU benchmark, ESP-NOW 4-node send loss rate, 2-path mixing stability, BOOT PTT boot conflict, standby current + wake latency, ECDH 6x X25519 timing, ESP-NOW peer table capacity (12 entries), channel evaluation accuracy. Total = 10 scenarios.

### D35: BootPress carries screen_was_off (affects: 04, 10)
- Change 04: update `InputEvent::BootPress` to carry `screen_was_off: bool` field. Update spec "InputEvent 枚举" requirement. This is a design refinement of the trait, not a MODIFIED capability (change 04 is where InputEvent is first defined).
- Change 10: use `BootPress { screen_was_off }` from change 04. Do NOT list as modifying InputEvent (it's defined in 04 with this field). Update design D8.

### D36: AppContext spec requirement (affects: 07)
Add spec requirement "AppContext 结构": `AppContext` SHALL hold `&StorageService`, `&DisplayService`, `&PowerService`, `&Settings` (current in-memory copy). `App::on_enter(&mut self, ctx: &AppContext)`. Settings App uses injected `ctx.settings` not `load_settings()` on every enter.

### D37: on_capture_frame takes &AudioFrame (affects: 05)
Change callback signature from `Fn(AudioFrame)` (by value) to `Fn(&AudioFrame)` (by reference). Pool of 2 frames allows double-buffering: callback borrows frame A while capture fills frame B. Update spec "AudioService trait" and "采集帧池" requirements.

### D38: Fixed attenuation 0.7 + soft limiter (affects: 05)
`submit_pcm` mixing: each route × 0.7 attenuation → sum → soft limiter (tanh-like compression, NOT hard clip). Update spec "接收混音播放" requirement and design D5. Remove "1/N" language.

### D39: on_recv replaceable (affects: 06)
Document in spec that `on_recv` uses `Mutex<Option<Box<dyn Fn(RecvEvent) + Send + Sync>>>` internally, allowing replacement by calling `on_recv` again (old callback dropped). Add scenario "回调替换".

### D40: VoiceEffect runtime-only (affects: 14)
Keep VoiceEffect as runtime memory only, not NVS. Add note in design: "VoiceEffect 不持久化到 NVS，重启后重置为 Normal。这是设计选择，非缺陷——变声器定位为趣味增强，非用户长期偏好。"

### D41: TD-PSOLA CPU risk + fallback (affects: 14)
Add to design Risks: "TD-PSOLA 在 160MHz RISC-V 单核上的 CPU 预算需实测。若单帧处理 >10ms（超过 20ms 帧长 50%），降级为简单重采样（线性插值，音质下降但无 CPU 风险）。" Update tasks to include benchmark spike.

### D42: ptt_release attribution fix (affects: 10)
Fix proposal.md: `network.send_unicast` comes from change 06 (NetworkService), not change 09. Update "Impact > 依赖" wording.

### D43: 13 attribution fix (affects: 13)
Fix proposal.md: `host_create`/`host_confirm`/`host_cancel`/`join_search_start`/`join_request`/`join_cancel` are from change 09 (intercom-pairing), not change 10. `ptt_press`/`ptt_release`/`set_voice_effect`/`preview_voice` are from change 10 (intercom-voice-ptt) / change 14 (voice-changer).

### D44: HEARTBEAT state field includes ChannelBusy (affects: 12)
HEARTBEAT packet `state` field: 0=Idle, 1=Talking, 2=Listening, 3=ChannelBusy. Note in spec that tech design §4.3 only lists 3 but ChannelBusy is the 4th VoiceState variant. Update spec "心跳包格式" requirement.

### D45: Screen-off PTT complementary (affects: 10, 15)
Document in both changes: change 10's `BootPress { screen_was_off: true }` triggers PTT without waking screen. Change 15's ScreenPolicy forwards `BootGpioPress` to InputService without calling `screen_on()`. Complementary, not conflicting. Add cross-reference in both designs.

### D46: MAX_WATER floor 6 (affects: 11)
Spec "per-sender 抖动缓冲": `max_water = max(6, observed + 2)`, hard-capped at 10. Add floor of 6 (was missing). Update spec.

### D47: Ring buffer fixed size 10 (affects: 11)
`ring: [FrameSlot; 10]` (hard cap = 10, compile-time fixed). `max_water` is a runtime variable ≤ 10 controlling when to drop oldest. Update tasks 2.2.

### D48: W2 AudioFrame pool double-buffer (affects: 05)
With D37 (callback takes &AudioFrame), pool of 2 frames now provides real double-buffering: capture writes frame A while callback reads frame B. Update design D7 to document this.

### D49: Settings App uses injected Settings (affects: 07)
Tasks 5.3: change from "load_settings() on on_enter" to "use ctx.settings from AppContext". Only call `load_settings()` on Launcher startup. Settings App reads from injected `&Settings`, writes via `save_settings()` + updates `ctx.settings`. Update tasks 5.3.

### D50: DirectoryBroadcast entry_at named return (affects: 08)
Spec "DIRECTORY_BROADCAST": `entry_at(&self, i: usize) -> Result<DirectoryEntry, PacketError>` where `DirectoryEntry { mac: &[u8;6], pub_key: &[u8;32] }` (named struct, not bare tuple). Update spec + tasks 10.2.
