# 2. 硬件平台与约束

## 2.1 主控与无线

| 项目 | 内容 | 约束 |
|---|---|---|
| MCU | ESP32-C6 RISC-V 单核 160MHz | 无 PSRAM |
| SRAM / Flash | 512KB / 16MB | — |
| 无线 | Wi-Fi 2.4G / BLE 5 / 802.15.4 | ESP-NOW 与 Wi-Fi 共用单射频 |
| 射频约束 | — | 对讲优先级 > 其他无线任务；同信道体系工作；实时音频与 UI 资源严格控制 |

## 2.2 显示与触摸

| 项目 | 内容 | 约束 |
|---|---|---|
| LCD | ST7789 240×240 SPI | 低信息密度 UI |
| 触摸 | CST816 | 熄屏首触只唤醒，不触发点击 |
| 背光 | 软件可控 | 支持熄屏唤醒 |

## 2.3 音频链路

| 项目 | 内容 | 约束 |
|---|---|---|
| 采集 | ES7210 | 单麦、单声道 |
| 播放 | ES8311 + NS4150B 功放 | 单声道外放 |
| 不做 | 双麦立体声 / 侧音 / 真全双工 / AEC / AGC / 波束成形 / 双麦降噪 | — |

## 2.4 电池与功耗

| 项目 | 内容 | 约束 |
|---|---|---|
| 供电 | 电池供电 | 省电靠熄屏/降频/按需启停音频链路/低功耗待机 |
| 电量估算 | ADC 估算（BAT_ADC=GPIO0，分压×3） | 平滑+滞回；避开发射/大音量瞬时采样 |
| 充电状态 | 不可靠，不作为正式能力 | — |
| PMIC | 无可编程独立控制；不支持软件真关机 | 第一期不做真关机、不做完整低电保护 |

## 2.5 按键

| 按键 | 角色 | 已知限制 |
|---|---|---|
| BOOT (GPIO9) | 主 PTT | 启动过程中不支持按住开机发言；正常启动后生效 |
| PLUS (GPIO18) | 全局音量入口 | — |
| PWR | 不参与普通软件交互 | 长按关机 |

# 3. 硬件导出的产品结论

- 不做双麦立体声 / 不做软件真关机
- 对讲模式不承诺固定 sleep 状态，仅承诺低功耗待机
- PWR 键不参与普通交互
- 音频必须压缩传输，禁用 PCM 直传
- 自由模式下本机不支持边说边听（单射频半双工）
- 各设备对信号/在线状态独立观测，显示允许不完全一致
- 组内通信以加密单播为主，不依赖加密广播
- **最大正式产品组容量：4 台设备**

# 4. 系统总体架构

## 4.1 架构原则

- 单固件、不动态加载；App 不直接访问硬件
- 能力通过 Service 提供；系统设置是全局设置
- 对讲专属逻辑不进全局 Settings
- 分层保留但实现轻量；不为未来过度工程化
- 运行时关键路径优先稳定和简单

## 4.2 分层结构

```text
Applications
├─ Intercom App（对讲）
└─ Settings App
App Shell / Launcher
├─ App 注册与切换 / 前台页面管理
├─ 全局状态栏 / 输入事件分发
└─ 全局熄屏策略 / 全局音量面板
Services
├─ Intercom Service / Audio Service
├─ Network Service / Display Service
├─ Input Service / Power Service / Storage Service
└─ Future Skeleton（Time / OTA / Wi-Fi / BLE）
Board Profile（轻量只读模块）
├─ 屏幕能力 / 音频能力 / 输入能力 / 默认参数
└─ Feature Flag
HAL / BSP
├─ LCD(ST7789) / Touch(CST816)
├─ Audio(ES7210 / ES8311 / PA)
├─ Battery ADC / Buttons
└─ ESP-NOW / Wi-Fi / BLE
```

## 4.3 Board Profile 职责

| 负责（只读） | 不负责（运行时状态） |
|---|---|
| 屏幕分辨率/背光调节、可用按键、单麦/单声道能力 | 在线状态、当前音量、前台 App |
| 是否真关机、电量读取方式、默认亮度/熄屏时间 | 网络连接状态、实时成员状态 |
| 最大组容量、Feature Flag | — |

# 5. 启动、恢复与安全回退

## 5.1 启动流程

```text
上电
→ 初始化 BSP / 基础服务
→ 读取系统设置（schema version 校验，失败回退默认）
→ 读取本地组信息（schema version 校验，失败/不兼容→清理）
→ 校验结果
   ├─ 无组信息 → 未组网
   └─ 有效组信息 → 重建 LMK 加密表，进入已组网待机
→ 启动 Launcher / 上次前台 App
```

## 5.2 前台恢复

| 场景 | 行为 |
|---|---|
| 无组关系 | 进入 Launcher |
| 有组关系 + 上次前台为对讲 | 进入对讲 App |
| 有组关系 + 上次前台非对讲 | 进入上次前台 App，后台保持对讲待机 |

## 5.3 安全启动回退

连续多次启动异常 → 进入安全启动模式：暂不恢复对讲服务；允许进入 Settings 清空组信息或恢复出厂。

## 5.4 存储结构版本（schema version）

- 系统设置与组信息均带 `schema version`
- 读取失败/字段非法：系统设置回退默认值；组信息直接清理
- **不保留空中协议版本字段**；组队兼容性由 `schema version` + 公钥总表格式隐式校验

# 7. 全局输入与交互

## 7.1 按键定义

| 按键 | 行为 |
|---|---|
| BOOT 按住 | PTT 发言 |
| BOOT 短触 / 普通场景 | 唤醒屏幕 |
| PLUS 短按 | 呼出全局音量面板 |
| PLUS 长按 | 全局静音 / 取消静音 |
| PWR | 不参与普通交互 |

## 7.2 PTT 流程

```text
按下 BOOT → 准备态（预热采集链路）→ 就绪提示音 → 用户讲话
→ 松开 BOOT → 停止发言 → 恢复接收/待机
```

| 细节要求 |
|---|
| 需按压触发阈值，避免误触短按直接发言 |
| 就绪提示音须在正式采集/发送前完成，加短保护间隔，不污染语音内容 |
| 熄屏按住 PTT → 直接发言，不亮屏 |
| 已组网 + 对讲服务可用时，BOOT 长按在任何前台 App 中均可作 PTT |

## 7.3 触摸规则

- 对讲前台页可提供大面积触摸 PTT 备用入口
- 熄屏首触只亮屏，不触发点击；亮屏后第二次触摸才进入实际操作

# 9. 电量与显示策略

## 9.1 电量

| 项 | 规则 |
|---|---|
| 显示 | 仅电量档位图标，不强推百分比；不显示充电中/已充满 |
| 估算 | 平滑后 ADC 值；避开发射/大音量瞬时采样；档位滞回避免跳变 |

## 9.2 自动熄屏

| 项 | 规则 |
|---|---|
| 默认熄屏时间 | 30 秒（可配置） |
| 熄屏后收到他人语音 | 不亮屏，正常播放 |
| 熄屏后触摸 | 首触只唤醒 |
| 熄屏后按 PTT | 不亮屏，直接发言 |

## 9.3 非对讲模式待机

进入低功耗待机须同时满足：系统空闲 / 无前台持续任务 / 无用户交互 / 不影响当前关键功能。

> 对讲模式下不牺牲实时收发能力以省电。

# 10. 对讲产品模型

## 10.1 阶段划分

```text
未组网 → 组队中 → 已组网
```

> Host/Join 角色仅存在于组队阶段；组队成功后全员平等。

## 10.2 组规则

| 规则 |
|---|
| 不区分 Host/Join、无群主、无管理员 |
| 不支持在线增员、不支持删除其他成员、不支持在线修改模式 |
| 成员变更必须整组重新组队 |
| 离线成员默认保留；心跳超时只标离线，不自动删除 |
| 要新增/正式移除成员，必须全员重新组队 |

## 10.3 容量

第一阶段产品上限：**4 台设备**（全网状 ECDH 下 4 节点 = 6 对 LMK）。

# 11. 组队与配对流程

> 本章整体采用 **三阶段 ECDH 全网状组队方案**。组队阶段保留 Host/Join 命名，组网成功后全员平等。

## 11.1 三阶段总览

```text
[阶段1] 1信道 · 星型公钥收集
   Host 明文广播 ← Join 单播响应
   双方互换本地动态 Curve25519 公钥
   建立点对点初步 LMK 通道
        ↓
[阶段2] 1信道 · 大广播通讯录
   Host 明文控制广播包下发：
   [ 全员 MAC ↔ 公钥映射总表 | 目标信道 | 统一切换时间戳 ]
        ↓
[阶段3] 切信道 · 全对等重构
   全员同步切至目标信道 → 转为完全平等 Peers
   各节点后台用【本机私钥 × 表中各对端公钥】ECDH 计算两两 LMK
   写入 esp_now_add_peer 硬件加密表 → 全网加密对讲打通
```

## 11.2 信道协商

| 阶段 | 信道 | 动作 |
|---|---|---|
| 发现 | 固定 1 信道 | Host 创建 + Join 搜索均在此 |
| 评估 | 1 信道 | Host 对候选信道做快速噪声/空闲度评估，选定最佳工作信道 |
| 切换 | 1 → 目标 | Host 连续多次广播「目标信道 + 统一切换时间戳/倒计时」，到点同步切 |
| 持久化 | 目标信道 | 组网成功后写入 NVS；重启优先恢复原信道 |
| 不支持 | — | 在线切换信道；链路长期差只能退出重组 |

## 11.3 Host 流程

```text
进入对讲（未组网）→ 创建主机 → 选择模式（清晰/自由）
→ 评估候选信道并选定工作信道 → 进入组队中（1信道）
→ 收集 Join 公钥 → 显示当前成员列表 → 确认组队
→ 冻结成员列表 → 大广播通讯录+切换时间戳
→ 全员切至目标信道 → 各节点重构 LMK → 进入已组网
```

## 11.4 Join 流程

```text
进入对讲（未组网）→ 搜索主机（1信道）→ 列表选择目标
→ 加入请求 → 单播互换公钥 → 查看模式/成员/人数/信号
→ 等待 Host 确认 → 收到通讯录+切换时间戳
→ 切至目标信道 → 重构与本组其他各节点 LMK → 进入已组网
```

## 11.5 搜索主机列表字段

| 字段 | 必显 | 说明 |
|---|---|---|
| 主机名称 | 是 | — |
| 轻量辅助标识 | 是 | **MAC 后 4 位**（避免重名误加） |
| 信号 4 格 | 是 | — |
| 当前模式 | 是 | 清晰/自由 |
| 当前人数 / 上限 | 是 | — |
| 是否可加入 | 是 | — |

> 主页面不显示辅助标识；搜索页/组队确认页必须保留。

## 11.6 组队超时与冻结

| 项 | 规则 |
|---|---|
| Host 组队超时 | 进入组队页后 60~120 秒未确认 → 自动结束，回未组网，提示"组队超时，已结束" |
| 冻结点 | Host 点击确认后立即冻结，不再接受新成员；当前成员完成最终同步后进入已组网 |

## 11.7 失败场景覆盖

| 场景 |
|---|
| 搜索不到主机 / 加入超时 / 主机已结束组队 / 人数已满 |
| schema version 不兼容 / 模式或状态变化导致失败 / 信号弱·链路不稳定 |

均需明确失败原因 + 重试入口。

# 12. 组信息持久化与恢复

## 12.1 持久化字段（NVS）

| 分类 | 字段 |
|---|---|
| 密钥材料 | 本机 Curve25519 私钥 |
| 对端信息 | 各对端 MAC ↔ 公钥映射总表 |
| 组配置 | 当前模式（清晰/自由）、当前工作信道 |
| 结构 | `schema version` |

> 废弃原 PRD 的 Group ID / Group Key / Group Epoch / Host·Join Nonce 等字段。

## 12.2 本地存储要求

- 第一期允许明文保存组信息与本机私钥
- 不要求应用层加密存储
- 允许设备在物理隔离场景中使用

## 12.3 重启恢复规则

| 场景 | 行为 |
|---|---|
| 组信息有效 | 读 NVS → 内存重跑 ECDH 恢复全套 LMK → 写入硬件加密表 → 进入组内待机。**全程无空中无线电交互** |
| 组信息损坏/不兼容 | 清理组信息，回未组网 |
| 系统设置损坏 | 回退默认设置 |

# 13. 对讲模式

## 13.1 对比

| 维度 | 清晰模式 | 自由模式 |
|---|---|---|
| 目标 | 传统对讲，秩序与清晰度 | 自由多人互动，趣味性 |
| 频道占用 | 一次仅一人占用 | 多人可同时按 PTT |
| 抢话冲突 | 其他人按 PTT 显示"频道忙"+忙态反馈 | 无冲突概念 |
| 本机发言时收听 | 不播放他人音频（半双工硬件约束） | 不播放他人音频（同左） |
| 接收端混音 | 单路 | 良好环境支持 **2 路**；best-effort 3 路（不作为刚性承诺） |
| 并发边界 | 极少数近同时抢话允许短时退化为近似自由模式，应尽快回单人占用 | — |
| UI | 高亮当前发言者 + 频道忙状态（视觉优先，音效辅助） | 逐个显示所有正在发言者 |

## 13.2 自由模式混音实现

- 各路混音前固定衰减
- 输出限幅保护
- 资源不足时按稳定性和持续性选择保留流
- UI 显示"正在发言"≠一定能同时完整听到其音频

# 14. 对讲语音状态机

## 14.1 三态模型

```text
        ┌──────────┐
        │   IDLE   │←─────────────┐
        └────┬─────┘              │
     PTT按下 │              收发完成 │
            ↓                     │
   ┌────────────────┐    ┌────────┴──────┐
   │   TALKING      │    │   LISTENING   │
   │ 采集+编码+发送  │    │ 接收+解码+播放 │
   │ 关闭接收链      │    │ 关闭采集链    │
   └───────┬────────┘    └───────────────┘
        PTT松开↓                 ↑收到他人语音
           └────────────────────┘
```

## 14.2 半双工互斥规则

| 本机状态 | 采集链 | 接收链 | 说明 |
|---|---|---|---|
| IDLE | 关 | 开（待机低功耗） | 等待 PTT 或他人语音 |
| TALKING | 开 | **关** | 单射频限制，只发不收 |
| LISTENING | **关** | 开 | 只收不发 |

## 14.3 自由模式本机策略

自由模式下本机 TALKING 时因硬件限制无法收听他人，但**不影响网络上其他 LISTENING 设备同时接收并混音另外几台设备的语音**。

# 15. 在线状态、心跳与信号

## 15.1 在线状态

- 每台设备独立维护其他成员在线状态
- 心跳超时仅标记离线，不自动删除
- 重新收到消息 → 重新标在线

## 15.2 信号显示

仅 4 格粗颗粒；做平滑+滞回避免频繁跳变；不显示精确数值。

## 15.3 心跳策略

| 状态 | 策略 |
|---|---|
| 已组网待机 | 低频 |
| 组队中 | 相对较高 |
| 正在稳定语音收发 | 降频（语音活动可辅助视为在线） |
| 熄屏低功耗待机 | 低频保持 |

> 组网后为去中心化 Peers 拓扑：接收到对端心跳即自适应更新该节点在线状态与 4 格信号。

# 16. 音频方案

## 16.1 能力边界

| 项 | 规则 |
|---|---|
| 采集/播放 | 单麦、单声道 |
| 不做 | 立体声录音 / 双麦处理 / 侧音 / 全双工 / AEC / AGC / 波束成形 |

## 16.2 编解码（Opus）

| 项 | 配置 |
|---|---|
| 算法 | Opus |
| 采样率 | 16kHz |
| 声道 | 单声道 |
| 帧长 | 20ms |
| 数制 | **定点数（Fixed-point）优化**，全面关闭浮点 API |
| 调整原则 | 若 CPU/RAM/包大小/稳定性不达标，可调更轻量配置；优先稳定可用 |

## 16.3 单包策略

- 一帧语音对应一个 ESP-NOW 包
- 一期默认不做语音分片
- 若某配置导致单帧超单包上限，不纳入一期正式支持

## 16.4 语音包时序与去重

| 项 | 规则 |
|---|---|
| 序号字段 | 包头含单调递增 Sequence Number |
| 旧包/重复包过滤 | 收到序号 ≤ 当前已处理最大序号 → 直接丢弃 |
| 不手动空白补偿 | 丢包/断帧时**绝不**填充物理静音或空白采样块 |

## 16.5 丢包隐藏（PLC）

| 项 | 规则 |
|---|---|
| 机制 | 复用 Opus 原生 PLC（向解码器塞 null 指针/触发丢包标记，由算法预测当前帧） |
| 静音封底 | 连续严重丢包 **超过 4 帧 / 80ms** 且算法失去预测基准时，才输出静音 |
| 目标 | 保障主观连续性，避免物理爆音 |

## 16.6 抖动缓冲（Jitter Buffer）

| 项 | 规则 |
|---|---|
| 初始水位 | 收首帧后不立即播放，建立 **3 帧（60ms）** 初始缓存 |
| 最大水位 | 动态上限，防止抖动累积导致延时无限拉长 |
| 时钟漂移 | 长时间接收需考虑设备间采样时钟漂移；轻量缓冲水位纠偏；优先主观连续性，不做重型重采样 |

## 16.7 弱网退化优先级

```text
1. 实时性 > 2. 音质下降 > 3. 限制混音路数 > 4. 抖动缓冲有上限 > 5. 极端情况局部丢字
```

> 弱网下优先保证实时性与可懂度，不追求播放器式完整回放。

## 16.8 音频链路启停

| 项 | 规则 |
|---|---|
| PTT 准备 | 预热采集链路，减少首字丢失 |
| 接收播放 | 避免包级频繁开关 PA；进入播放前允许极短启动缓冲 |
| 爆音抑制 | 启停需避免爆音、喀哒声、尾音硬切 |

## 16.9 实现约束

- 实时音频与网络路径不得依赖高频动态内存分配
- 采集/编码/发送/接收/解码/混音/播放采用预分配固定缓冲与对象池

# 18. 安全模型（ECDH 全网状加密）

## 18.1 安全目标

| 目标 | 达成 |
|---|---|
| 非组内设备不能正常加入通话 | ✓ |
| 非组成员不能正常解码组内语音/信令 | ✓（防空中被动监听） |
| 重启后可恢复组关系 | ✓（NVS 冷启动免空中交互） |
| 交互简单 | ✓ |

## 18.2 安全边界

| 不承诺 |
|---|
| 强抗中间人（MitM）攻击 |
| 验证码 / SAS 配对 |
| 强意义上单设备吊销 |
| 成员退出即密码学失效 |

> 单成员正常退出仅清理本机 NVS；设备遗失/强制剔除成员必须全员解散并在 1 信道重新走公钥交换以轮换全新随机密钥对。

## 18.3 密钥模型

| 项 | 规则 |
|---|---|
| 硬编码密钥 | **彻底废除**固件硬编码 Root Key 与 GroupKey |
| 唯一密钥类型 | 全生命周期仅维护一套两两对等 LMK（Link Master Key） |
| 算法 | Curve25519 ECDH |
| 生成时机 | 组队时各节点用硬件随机数生成临时公私钥 |
| LMK 计算 | 【本机私钥 × 各对端公钥】ECDH → 16 字节 LMK → 写入硬件加密表 |
| 拓扑 | 全网状（Full Mesh），4 节点 = 6 对独立 LMK |

## 18.4 三阶段组队加密流程

```text
阶段1 · 1信道 · 星型公钥收集
   Host 明文广播 ← Join 单播响应
   互换本地动态 Curve25519 公钥
   初步 LMK 通道建立（Host↔各 Join 点对点）
        ↓
阶段2 · 1信道 · 大广播通讯录
   Host 明文控制广播包下发：
   [ 全员 MAC ↔ 公钥映射总表 | 目标信道 | 统一切换时间戳 ]
        ↓
阶段3 · 目标信道 · 全对等重构
   全员切信道 → 转为平等 Peers
   各节点后台：【本机私钥 × 表中各对端公钥】ECDH → 两两 LMK
   esp_now_add_peer 硬件加密注册 → 全网加密对讲打通
```

## 18.5 运行时加密方式

组网成功后全部加密单播，应用层不再对音频流做额外重加密：

| 业务 | 方式 |
|---|---|
| 音频包 / 心跳包 / 发言状态 / 控制消息 | 加密单播 |

## 18.6 持久化与冷启动恢复

| 项 | 规则 |
|---|---|
| 迁移工作信道前 | 强制写 NVS：本机私钥 + 对端公钥/MAC 总表 + 当前信道 + 模式 |
| 冷启动 | 读 NVS → 内存重跑 ECDH 恢复全套 LMK → 写入硬件 → 就绪。**全程无空中无线电交互，瞬时恢复** |
| 正常退出 | 仅清理本机 NVS |
| 设备遗失/剔除 | 全员解散 → 1 信道重新公钥交换 → 轮换全新随机密钥对 |

# 22. 状态机

## 22.1 系统状态机

```text
开机 → 初始化系统服务 → 读取系统设置 → 读取组信息
→ 校验 schema version
   ├─ 无效 → 清理 → 未组网
   └─ 有效 → 重建 LMK 加密表 → 已组网待机
→ 进入 Launcher / 上次前台 App
```

## 22.2 对讲状态机（含语音三态）

```text
未组网
├─ 创建主机 → 组队中
├─ 搜索主机并加入 → 组队中
└─ 返回系统

组队中
├─ 三阶段完成（切信道+重构 LMK） → 已组网
├─ 加入者退出 → 未组网
├─ 主机结束组队 → 未组网
├─ 超时/失败 → 未组网

已组网
├─ 按下 PTT → TALKING
│   ├─ 清晰模式+频道空闲 → 发言中
│   ├─ 清晰模式+频道忙 → 忙态提示
│   └─ 自由模式 → 发言中
├─ 收到他人语音 → LISTENING
├─ 熄屏 → 低功耗待机（保持对讲可用）
└─ 主动退出 → 未组网
```

## 22.3 成员状态机

```text
已组网
├─ 心跳正常 → 在线
├─ 心跳超时 → 离线
└─ 重新收到消息 → 在线
```

# 23. 异常与边界场景

| 场景 | 行为 |
|---|---|
| 清晰模式近同时按 PTT | 允许短时退化为近似自由模式效果，尽快回单人占用（属已知可接受退化，非缺陷） |
| 熄屏收到他人语音 | 不亮屏，正常播放；手动唤醒后可看最新状态 |
| 熄屏按 PTT | 直接发言，不亮屏，维持省电策略 |
| 成员长期离线 | 仅显示离线，不自动删除；正式移除需整组重建 |
| 全局静音开启 | 页面持续显示静音图标；语音/提示/预览均受影响；保留视觉状态反馈 |
| schema version 不兼容 | 不允许加入；本地旧组信息直接作废并清理 |
| 数据损坏 | 系统设置回退默认；组信息清理；不得卡死 |

# 26. 验收标准

## 26.1 系统架构

- 具备 App + Service + BSP 分层；Board Profile 轻量只读
- 对讲与 Settings 明确分离；未实现功能不暴露空入口

## 26.2 组队（三阶段 ECDH）

- A 创建主机并选模式；B 在 1 信道搜索到 A
- B 加入过程可看模式/成员列表/人数/信号/MAC后4位辅助标识
- A 确认组队后列表冻结；通讯录+切换时间戳下发
- 全员切目标信道；各节点后台 ECDH 重构两两 LMK；加密对讲打通
- A/B 本地 NVS 持久化私钥+公钥总表+信道+模式
- 组队成功后不再显示 Host/Join 身份
- 重启后各成员读 NVS 免空中交互恢复组关系

## 26.3 模式

**清晰模式**：正常同时刻仅一人发言；其他人按 PTT 收到忙态反馈；近同时竞争场景允许短时退化但应恢复单人占用。

**自由模式**：A 与 B 可同时发言；C 在良好环境可混音接收至少 2 路远端语音；UI 逐个显示发言成员；本机发言时不播放他人音频。

## 26.4 退出与成员状态

- 本机退出后清理本地 NVS（私钥+公钥总表+组信息）
- 其他成员后续收不到其消息 → 心跳标离线
- 离线成员不自动删除
- 成员变化需整组重建；设备遗失/剔除必须全员解散并 1 信道重新公钥交换轮换密钥

## 26.5 设置

- 可设设备名称、随机生成名称、全局音量、静音、亮度、自动熄屏时间
- 恢复出厂后清空所有本地组信息和设置

## 26.6 功耗与熄屏

- 默认 30 秒自动熄屏生效
- 对讲模式熄屏后仍能继续收听与发言
- 熄屏按 PTT 不亮屏；收到他人语音不亮屏
- 触摸首触只唤醒

## 26.7 安全

- 未组网设备不能正常加入现有组
- 非组成员不能正常解码组内通信（防空中被动监听）
- 重启后组关系可恢复（NVS 冷启动免空中交互）
- 单成员退出≠密码学吊销；整组重建后旧 LMK 全部失效

## 26.8 稳定性与长稳

- 2 台设备长时间待机后仍可通话
- 反复熄屏/唤醒后仍正常
- 连续重启恢复多次后组关系正确
- 异常数据不卡死
- 多设备长时间在线时成员状态稳定显示

# 28. 风险

| 风险 | 说明 |
|---|---|
| Opus 集成 | Rust + ESP-IDF + 无 PSRAM，需尽早验证定点数优化下 CPU/RAM 占用 |
| ESP-NOW 语音实时性 | 多设备+加密+音频并行场景需压力测试 |
| 自由模式混音 | 2 路稳妥，3 路 best-effort |
| BOOT 作 PTT | 开发板阶段折中方案 |
| 功耗 | 熄屏低功耗待机与保持实时对讲需实机验证 |
| 声学 | 外放不支持边说边听，避免啸叫与回灌 |
| 清晰模式并发边界 | 轻仲裁下极少数并发场景短时退化 |
| ECDH 全网状 | 4 节点 6 对 LMK，需验证 ESP-NOW peer 表容量与加密表写入时序 |

# 29. 成功标准

- 用户能稳定完成组队（三阶段流程）
- 所有成员都能本地保存并冷启动免空中恢复组关系
- 对讲在熄屏低功耗待机下仍保持可用
- PTT 使用简单直接
- 清晰模式正常情况下有秩序，自由模式具备趣味性
- 主页面信息简单清晰，适合快速扫视
- 弱网下优先保实时与可懂度
- 全局设置真正服务于整体系统
- 后续新增功能时不需重构系统底座
