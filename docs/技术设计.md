```text
│  ESP-NOW/Wi-Fi/BLE                          │
└─────────────────────────────────────────────┘
```

## 1.2 模块依赖关系

```text
Intercom App ──→ Intercom Service ─┬─→ Audio Service
                                   ├─→ Network Service ──→ Crypto Service
                                   ├─→ Storage Service
                                   └─→ Power Service
Settings App ──→ Storage Service / Display Service / Power Service
Launcher ─────→ App 注册表 / Input Service / Display Service
```

依赖方向单向自上而下；Service 之间通过 trait 接口耦合，不直接持有具体类型。

## 1.3 关键设计原则

- App 不直接访问硬件，全部通过 Service
- 实时音频与网络路径不得依赖高频动态内存分配；全链路使用预分配固定缓冲与对象池
- 关键路径优先稳定和简单；不为了未来预留而过度工程化
- 系统设置是全局的；对讲专属配置不进 Settings

## 3.2 Crypto Service

```rust
pub trait CryptoService: Send + Sync {
    /// 生成临时 Curve25519 公私钥对
    fn gen_keypair(&self) -> KeyPair;

    /// ECDH: 本机私钥 × 对端公钥 → 16 字节 LMK
    /// 内部做 Curve25519 标量乘 + KDF 截断为 16 字节
    fn derive_lmk(&self, my_priv: &[u8; 32], peer_pub: &[u8; 32]) -> [u8; 16];

    /// 校验公钥合法性（点在曲线上、非全零）
    fn validate_pubkey(&self, pub_key: &[u8; 32]) -> bool;
}

pub struct KeyPair {
    pub priv_key: [u8; 32],
    pub pub_key: [u8; 32],
}
```

## 3.3 Network Service（ESP-NOW）

```rust
pub trait NetworkService: Send + Sync {
    fn init(&self, channel: u8) -> Result<(), NetError>;
    fn set_channel(&self, ch: u8) -> Result<(), NetError>;

    /// 注册/更新 peer（含 LMK 写入硬件加密表）
    fn add_peer(&self, mac: &[u8; 6], lmk: &[u8; 16]) -> Result<(), NetError>;
    fn remove_peer(&self, mac: &[u8; 6]) -> Result<(), NetError>;
    fn clear_peers(&self) -> Result<(), NetError>;

    /// 发送加密单播（payload 由 Intercom 层封装）
    fn send_unicast(&self, dst: &[u8; 6], payload: &[u8]) -> Result<(), NetError>;

    /// 发送明文广播（仅组队阶段使用，信道 1）
    fn send_broadcast(&self, payload: &[u8]) -> Result<(), NetError>;

    /// 注册接收回调（回调内禁止阻塞，应投递到队列）
    fn on_recv(&self, cb: Box<dyn Fn(RecvEvent) + Send + Sync>);

    /// 评估候选信道 RSSI/空闲度，返回推荐工作信道
    fn evaluate_channels(&self, candidates: &[u8]) -> u8;

    fn get_rssi(&self) -> i8;
}

pub struct RecvEvent {
    pub src_mac: [u8; 6],
    pub payload: Vec<u8>,    // 接收时已由 ESP-NOW 硬件解密
    pub rssi: i8,
}

pub enum NetError { Init, Channel, PeerFull, TxFail, Rx }
```

# 4. ESP-NOW 包格式

## 4.1 通用包头（8 字节）

| 偏移 | 长度 | 字段 | 说明 |
|---|---|---|---|
| 0 | 1 | magic | 固定 0xC6，用于过滤非本协议包 |
| 1 | 1 | ver | schema 版本（与 NVS 同步） |
| 2 | 1 | type | 包类型（见下表） |
| 3 | 1 | flags | 保留位 |
| 4 | 2 | seq | 单调递增序号（大端），用于语音去重 |
| 6 | 2 | len | payload 长度（大端） |

### 包类型 type 取值

| 值 | 类型 | 阶段 | 加密 |
|---|---|---|---|
| 0x01 | VOICE | 已组网 | 加密单播 |
| 0x02 | HEARTBEAT | 已组网 | 加密单播 |
| 0x03 | TALK_STATE | 已组网 | 加密单播 |
| 0x04 | CTRL_BUSY / CTRL_READY | 已组网 | 加密单播 |
| 0x10 | PAIR_BEACON_HOST | 阶段1 | 明文广播 |
| 0x11 | PAIR_JOIN_REQ | 阶段1 | 明文单播 |
| 0x12 | PAIR_JOIN_ACK | 阶段1 | 明文单播 |
| 0x20 | DIRECTORY_BROADCAST | 阶段2 | 明文广播 |
| 0x30 | CHANNEL_SWITCH_ACK | 阶段3 | 加密单播 |

## 4.2 VOICE 包（type=0x01）

| 偏移 | 长度 | 字段 | 说明 |
|---|---|---|---|
| 0 | 8 | header | seq 为语音帧序号 |
| 8 | 1 | sender_id | 发言者本地 id（成员表索引，0-3） |
| 9 | 1 | effect | 变声档位（接收端不处理，仅展示） |
| 10 | N | opus_payload | 一帧 Opus 数据，N ≤ 单包上限 |

> 单包策略：一帧语音 = 一个 ESP-NOW 包。若某 Opus 配置使单帧超 ESP-NOW 单包上限（约 250B），不纳入一期支持。

## 4.3 HEARTBEAT 包（type=0x02）

| 偏移 | 长度 | 字段 | 说明 |
|---|---|---|---|
| 0 | 8 | header | seq 心跳序号 |
| 8 | 1 | sender_id | — |
| 9 | 1 | state | 本机 VoiceState（Idle/Talking/Listening） |
| 10 | 1 | mode | 清晰/自由 |

## 4.4 TALK_STATE 包（type=0x03）

| 偏移 | 长度 | 字段 | 说明 |
|---|---|---|---|
| 0 | 8 | header | — |
| 8 | 1 | sender_id | — |
| 9 | 1 | action | 1=开始发言, 0=结束发言 |

> 清晰模式用于仲裁频道占用；自由模式用于 UI 展示。

## 4.5 CTRL_BUSY / CTRL_READY 包（type=0x04）

| 偏移 | 长度 | 字段 | 说明 |
|---|---|---|---|
| 0 | 8 | header | — |
| 8 | 1 | dst_id | 目标成员 |
| 9 | 1 | code | 1=CHANNEL_BUSY, 2=PTT_READY |

# 5. 三阶段组队协议

## 5.1 阶段总览

```text
阶段1 · 1信道 · 星型公钥收集
  Host: 周期广播 PAIR_BEACON_HOST（含本机 MAC+公钥+模式+当前人数+可加入）
  Join: 收到 beacon → 单播 PAIR_JOIN_REQ（含本机 MAC+公钥）
  Host: 单播 PAIR_JOIN_ACK（含本机公钥）
  → Host↔各 Join 建立"初步 LMK"（仅 Host↔Join，Join 之间尚未互通）

阶段2 · 1信道 · 大广播通讯录
  Host 点击确认组队 → 列表冻结
  Host 连续 N 次广播 DIRECTORY_BROADCAST（含全员 MAC↔公钥总表+目标信道+切换时间戳）

阶段3 · 目标信道 · 全对等重构
  到达时间戳 → 全员同步切目标信道
  各节点后台：对本组所有其他节点用【本机私钥×对端公钥】ECDH 派生两两 LMK
  调用 NetworkService::add_peer 写入硬件加密表
  发送 CHANNEL_SWITCH_ACK 给 Host（可选，用于确认全员到达）
  → 进入已组网待机
```

## 5.2 PAIR_BEACON_HOST 包（type=0x10，明文广播）

| 偏移 | 长度 | 字段 | 说明 |
|---|---|---|---|
| 0 | 8 | header | — |
| 8 | 6 | host_mac | — |
| 14 | 32 | host_pub_key | Curve25519 公钥 |
| 46 | 1 | mode | 清晰/自由 |
| 47 | 1 | cur_members | 当前已收集人数 |
| 48 | 1 | max_members | 上限 4 |
| 49 | 1 | joinable | 0/1 |

## 5.3 PAIR_JOIN_REQ 包（type=0x11，明文单播）

| 偏移 | 长度 | 字段 | 说明 |
|---|---|---|---|
| 0 | 8 | header | — |
| 8 | 6 | join_mac | — |
| 14 | 32 | join_pub_key | — |
| 46 | 6 | host_mac | 目标主机 MAC |

## 5.4 PAIR_JOIN_ACK 包（type=0x12，明文单播）

| 偏移 | 长度 | 字段 | 说明 |
|---|---|---|---|
| 0 | 8 | header | — |
| 8 | 6 | host_mac | — |
| 14 | 32 | host_pub_key | — |
| 46 | 6 | join_mac | 目标 Join MAC |
| 52 | 1 | accepted | 0/1 |
| 53 | 1 | reason | 失败码（满员/不兼容/状态变化） |

## 5.5 DIRECTORY_BROADCAST 包（type=0x20，明文广播）

| 偏移 | 长度 | 字段 | 说明 |
|---|---|---|---|
| 0 | 8 | header | — |
| 8 | 1 | member_count | N（含 Host） |
| 9 | 1 | mode | 清晰/自由 |
| 10 | 1 | target_channel | 选定工作信道 |
| 11 | 4 | switch_ts | 统一切换时间戳（本地 mono 时钟 ms，大端） |
| 15 | N×38 | entries | 成员条目数组 |

成员条目（38 字节/项）：

| 偏移 | 长度 | 字段 |
|---|---|---|
| 0 | 6 | mac |
| 6 | 32 | pub_key |

> Host 在切换时间戳前 200~500ms 区间连续广播 3~5 次该包，提高到达率。

## 5.6 CHANNEL_SWITCH_ACK 包（type=0x30，切信道后加密单播）

| 偏移 | 长度 | 字段 | 说明 |
|---|---|---|---|
| 0 | 8 | header | — |
| 8 | 1 | sender_id | 在新组的本地 id |
| 9 | 1 | status | 0=到达成功, 非0=失败码 |

> 用于 Host 确认全员到达目标信道。超时未收到某成员 ACK 则该成员在本机标离线，但组关系仍保留。

# 6. ECDH 全网状加密

## 6.1 LMK 计算流程

```text
输入：my_priv (32B), peer_pub (32B)
  ↓
Curve25519 标量乘：shared = X25519(my_priv, peer_pub)  → 32B
  ↓
KDF 截断：lmk = HKDF-SHA256(shared, salt="ESP32C6-INTERCOM", info="LMK-v1")[0..16]
  ↓
输出：lmk (16B) → 写入 ESP-NOW peer 表
```

> 16 字节对齐 ESP-NOW 硬件 AES-CCM LMK 长度要求。

## 6.2 4 节点全网状 LMK 表

| 节点对 | LMK | 写入位置 |
|---|---|---|
| A↔B | LMK_AB | A 的 peer[B] / B 的 peer[A] |
| A↔C | LMK_AC | A 的 peer[C] / C 的 peer[A] |
| A↔D | LMK_AD | A 的 peer[D] / D 的 peer[A] |
| B↔C | LMK_BC | B 的 peer[C] / C 的 peer[B] |
| B↔D | LMK_BD | B 的 peer[D] / D 的 peer[B] |
| C↔D | LMK_CD | C 的 peer[D] / D 的 peer[C] |

4 节点共 6 对 LMK；每节点需注册 3 个 peer。

## 6.3 esp_now_add_peer 调用时序

```text
// 切到目标信道后，每个节点本地执行：
for peer in directory.peers (除自己):
    lmk = crypto.derive_lmk(my_priv, peer.pub_key)
    network.add_peer(peer.mac, lmk)
    // 内部：esp_now_add_peer_with_lmk(mac, channel, lmk)
```

## 6.4 冷启动恢复流程

```text
上电
→ storage.load_group() → Option<GroupInfo>
→ 若 Some(g):
    network.init(g.channel)              // 直接初始化到工作信道（不经信道1）
    for peer in g.peers:
        lmk = crypto.derive_lmk(g.my_priv_key, peer.pub_key)
        network.add_peer(peer.mac, lmk)
    intercom.enter_grouped_idle()
→ 全程无空中无线电交互，瞬时恢复就绪
```

## 6.5 安全边界

| 项 | 承诺 |
|---|---|
| 防空中被动监听 | ✓（ECDH + ESP-NOW AES-CCM 硬件加密） |
| 抗 MitM | ✗ |
| 成员退出即密码学吊销 | ✗（仅清本机 NVS） |
| 设备遗失/剔除 | 必须全员解散 → 1 信道重新公钥交换 → 轮换全新密钥对 |

# 7. NVS 存储布局

## 7.1 命名空间划分

| Namespace | 用途 | 触发清理 |
|---|---|---|
| `sys` | 系统设置 | 恢复出厂 |
| `group` | 组信息（密钥+对端表+模式+信道） | 退出群组/恢复出厂/损坏 |
| `diag` | 诊断信息 | 恢复出厂 |

## 7.2 `sys` 命名空间

## 7.5 schema_ver 兼容性规则

| 场景 | 处理 |
|---|---|
| NVS schema_ver == 固件当前 ver | 正常加载 |
| NVS schema_ver < 固件 ver | 系统设置回退默认；组信息清理 |

## 9.1 系统状态机

```text
开机 → 初始化 → [校验]
                  ├─ 异常 → 安全启动模式
                  └─ 正常 → [组信息]
                              ├─ 无 → 未组网
```

## 9.2 对讲状态机

```text
未组网
├─ host_create → Hosting(Discovering) → Hosting(CollectingPeers)
├─ join_search_start → Joining(Searching) → join_request → Joining(Requesting)
└─ 返回系统

Hosting(CollectingPeers)
├─ host_confirm → Hosting(Frozen) → 阶段2广播 → Hosting(SwitchingChannel)
│                → 阶段3完成 → Grouped(Idle)
├─ host_cancel / 超时 → 未组网

Joining(Requesting)
├─ 收到 DIRECTORY_BROADCAST → Joining(SwitchingChannel) → 阶段3重构 → Grouped(Idle)
├─ join_ack accepted=0 → 未组网（失败原因提示）
├─ 超时 → 未组网

Grouped(VoiceState)
├─ ptt_press → Grouped(Talking)
│   ├─ 清晰+空闲 → 发言中
│   ├─ 清晰+忙 → Grouped(ChannelBusy)（不进 Talking）
│   └─ 自由 → 发言中
├─ 收到他人语音 → Grouped(Listening)
├─ 熄屏 → Grouped(Idle) + 低功耗待机（保持对讲可用）
└─ leave_group → 清 NVS → 未组网
```

## 9.3 语音三态机（Grouped 内部）

```text
        ┌──────────┐
        │   Idle   │←──────────────┐
        └────┬─────┘               │
   ptt_press│              收发完成 │
            ↓                     │
   ┌────────────────┐    ┌────────┴──────┐
   │   Talking      │    │   Listening   │
```

# 10. 音频流水线

## 10.1 发送链（TALKING）

```text
麦克风(ES7210,16kHz,mono) → 采集缓冲(20ms帧)
→ [变声处理(可选)]
→ Opus 编码(定点数) → 封装 VOICE 包(seq+sender_id+effect+opus)
→ Network.send_unicast(对每个组内 peer) → ESP-NOW AES-CCM 加密发送
```

## 10.2 接收链（LISTENING）

```text
ESP-NOW 收回调 → 解密 → 解析 VOICE 包
→ 序号去重(seq ≤ 已处理最大 → 丢弃)
→ 按 sender_id 分流 → 各路独立 Jitter Buffer(初始 3 帧/60ms)
→ 到达水位后开始解码：
    正常帧 → Opus 解码
    丢包(超时未到) → Opus PLC(null 指针触发预测)
    连续丢包 > 4 帧/80ms → 静音封底
→ 各路 PCM 固定衰减 → 混音 → 限幅 → ES8311 → NS4150B → 喇叭
```

## 10.3 资源池约束

| 项 | 规则 |
|---|---|
| 内存 | 采集/编码/发送/接收/解码/混音/播放全链路预分配固定缓冲与对象池 |
| 动态分配 | 实时路径禁止高频动态分配 |
| 混音路数 | 良好环境 2 路；best-effort 3 路；资源不足按稳定性保留流 |
| PA 控制 | 避免包级频繁开关；进入播放前极短启动缓冲；启停做爆音抑制 |

## 10.4 Opus 配置

| 项 | 值 |
|---|---|
| 采样率 | 16000 Hz |
| 声道 | 1（mono） |
| 帧长 | 20 ms |
| 数制 | 定点数（FIXED_POINT），关闭浮点 API |
| 比特率 | 实测调优，优先稳定 |

## 10.5 抖动缓冲

| 项 | 规则 |
|---|---|
| 初始水位 | 3 帧（60ms） |
| 最大水位 | 动态上限，防抖动累积致延时无限拉长 |
| 水位纠偏 | 轻量缓冲水位纠偏；优先主观连续性，不做重型重采样 |
| 时钟漂移 | 长时间接收考虑设备间采样时钟漂移 |

## 10.6 弱网退化优先级

```text
1. 实时性 > 2. 音质下降 > 3. 限制混音路数
4. 抖动缓冲有上限 > 5. 极端情况局部丢字（不用长缓存换完整性）
```

# 11. PTT 流程时序

```text
用户按住 BOOT
→ InputService::BootPress
→ IntercomService::ptt_press()
→ state = Grouped(Talking)
   ├─ 清晰模式：先发 TALK_STATE(action=开始) → 等待 50ms 仲裁窗口
   │   ├─ 期间收到他人 TALK_STATE → state=ChannelBusy, 给忙态反馈, 不进 Talking
   │   └─ 无冲突 → 进 Talking
   └─ 自由模式：直接进 Talking
→ AudioService::start_capture()（预热采集链，减少首字丢失）
→ 给就绪提示音（在采集前完成，加保护间隔）
→ 开始采集 → 编码 → 发送
→ 用户松开 BOOT
→ InputService::BootRelease
→ IntercomService::ptt_release()
→ 停止采集 → 发送 TALK_STATE(action=结束)
→ state = Grouped(Idle)
→ AudioService::stop_capture()
```

### 关键细节

- 按压触发阈值：BOOT 按下 ≥ 50ms 才视为 PTT（避免误触短按）
- 熄屏按 PTT：直接进 Talking，不亮屏，维持省电策略
- 已组网 + 对讲服务可用时，BOOT 长按在任何前台 App 都作 PTT

# 19. 关键路径伪代码

## 19.1 组队三阶段（Host 侧）

```rust
// 阶段1
fn host_create(mode) {
    network.init(DISCOVERY_CHANNEL); // 1
    let kp = crypto.gen_keypair();
    self.my_priv = kp.priv_key;
    self.my_pub = kp.pub_key;
    loop broadcast PAIR_BEACON_HOST every 200ms;
    on recv PAIR_JOIN_REQ from j:
        if members.len() < MAX && validate_pubkey(j.pub):
            members.push(j);
            send PAIR_JOIN_ACK(j, accepted=1);
            // 建立初步 LMK（仅 Host↔Join）
            let lmk = crypto.derive_lmk(self.my_priv, j.pub_key);
            network.add_peer(j.mac, lmk);
}

// 阶段2
fn host_confirm() {
    self.frozen = true;
    let target = network.evaluate_channels([1,6,11,13]);
    let ts = now_mono() + 500ms;
    for _ in 0..5 {
        broadcast DIRECTORY_BROADCAST(members, target, ts);
        sleep(80ms);
    }
    // 阶段3 触发点
    schedule_at(ts, || switch_channel(target));
}

// 阶段3
fn switch_channel(target) {
    network.clear_peers();              // 清除阶段1仅 Host↔Join 的初步 LMK
    network.set_channel(target);
    for peer in members (except self):
        let lmk = crypto.derive_lmk(self.my_priv, peer.pub_key);
        network.add_peer(peer.mac, lmk);  // 重构全网状
    storage.save_group(&group_info);     // 持久化私钥+公钥总表+信道+模式
    state = Grouped(Idle);
}
```

## 19.2 组队三阶段（Join 侧）

```rust
fn join_search_start() {
    network.init(DISCOVERY_CHANNEL);
    on recv PAIR_BEACON_HOST from h:
        ui.show_host(h);   // 名称+MAC后4位+信号+人数+模式+可加入
}

fn join_request(host_mac) {
    let kp = crypto.gen_keypair();
    self.my_priv = kp.priv_key;
    self.my_pub = kp.pub_key;
    send PAIR_JOIN_REQ(host_mac, self.my_pub);
    on recv PAIR_JOIN_ACK:
        if accepted:
            // 初步 LMK Host↔self
            let lmk = crypto.derive_lmk(self.my_priv, host_pub);
            network.add_peer(host_mac, lmk);
            state = Joining(WaitingConfirm);
}

// 阶段2 收到 DIRECTORY_BROADCAST
fn on_directory(dir) {
    self.directory = dir;
    schedule_at(dir.switch_ts, || switch_channel(dir.target_channel));
}

// 阶段3
fn switch_channel(target) {
    network.clear_peers();
    network.set_channel(target);
    for peer in directory.peers (except self):
        let lmk = crypto.derive_lmk(self.my_priv, peer.pub_key);
        network.add_peer(peer.mac, lmk);
    send CHANNEL_SWITCH_ACK(status=0);
    storage.save_group(&group_info);
    state = Grouped(Idle);
}
```

## 19.4 PTT 发言循环

```rust
fn ptt_press() {
    if state != Grouped(Idle): return;
    if mode == Clear {
        send TALK_STATE(action=开始);
        let conflict = wait_for_conflict(50ms); // 期间收到他人 TALK_STATE?
        if conflict {
            state = Grouped(ChannelBusy);
            ui.show_busy();
            play_busy_tone();
            return;
        }
    }
    state = Grouped(Talking);
    audio.pa_enable(true);
    audio.start_capture()?;   // 预热，减少首字丢失
    play_ready_tone();        // 在采集前完成 + 保护间隔
    audio.on_capture_frame(|f| {
        let pkt = build_voice_pkt(f.seq, self.id, self.effect, f.opus_data);
        for peer in group.peers (except self):
            network.send_unicast(peer.mac, &pkt);
    });
}

fn ptt_release() {
    audio.stop_capture();
    audio.pa_enable(false);
    if mode == Clear { send TALK_STATE(action=结束); }
    state = Grouped(Idle);
}
```

## 19.5 接收播放循环

```rust
fn on_recv_voice(pkt) {
    if pkt.seq <= last_seen_seq[pkt.sender_id] { return; } // 旧包/重复包丢弃
    last_seen_seq[pkt.sender_id] = pkt.seq;
    let buf = jitter[pkt.sender_id].push(pkt);
    if buf.waterline() < 3 { return; }   // 初始水位未达
    if buf.waterline() > MAX_WATER { drop_oldest(buf); } // 防延时累积
    let frame = buf.pop();
    let pcm = if frame.is_lost() {
        if consecutive_lost[sender] > 4 {
            consecutive_lost[sender] = 0;
            silence_block()   // 静音封底
        } else {
            consecutive_lost[sender] += 1;
            opus.decode(null)   // PLC 预测
        }
    } else {
        consecutive_lost[sender] = 0;
        opus.decode(frame.opus_data)
    };
    audio.submit_pcm(pkt.sender_id, &pcm);  // 各路固定衰减 → 混音 → 限幅 → 播放
}
```
